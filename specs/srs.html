Introduction

1.1 Purpose

The purpose of this document is to provide a description of the CISC 3140 group project, the Whack-A-Prof game. This document will cover the games features, where and how it will run, an overview of the game window, and desired graphics for the game.


1.2 Document Conventions

This document may contain terms that may be unfamiliar. Refer to Appendix A for terms and their respective definitions.


1.3 Intended Audience and Reading Suggestions

This SRS is intended for all members of Group 2. Members of the graphics team may want to refer to section 3 to view how the game window should look.


1.4 Project Scope

The Whack-A-Prof game is a browser based game that has the play style similar to that of Whack-A-Mole. The game play of Whack-A-Mole is described by the Wikipedia page in References.


1.5 References

Whack-A-Mole gameplay

Overall Description

2.1 Product Perspective

This game is a play on a classic carnival game called Whack-a-mole. In the image above one can see 7 holes for a mole to come out and a mallet/hammer.

  • Hole - A location for the mole to hide and come out of at random times
  • Mole - A target to be hit with a mallet. In Whack-A-prof the targets will be professors, dept heads, and trustees
  • Mallet/hammer - Used to hit a mole when it comes out from one for the 7 holes in the image. In Whack-A-Prof the mallet will be used to hit professors, dept heads, and trustees


2.2 Product Functions

  • Mole - The targets smoothly show up and hide in “holes”
  • Mallet
    • Move downwards when attempting to hit something and back upwards afterwards
    • Show contact animation when mallet hits something

2.3 User Classes and Characteristics

The Whack-A-Prof game will use Player class only. There will not be Administrator class.


2.4 Operating Environment

Minimum Versions:

  • Chrome: 65.0.3325.181 (Windows, Mac, Linux), 65.0.3325.152(iOS), 65.0.3325.109(Android)
  • FireFox: 58.0
  • Internet Explorer: 11(Windows 7 and up)
  • Safari: 11.0


2.5 Design and Implementation Constraints

The Whack-A-Prof game should run in a internet browser (i.e. - Google Chrome, Mozilla Firefox, etc.). The game is to be coded in Javascript, the animations to be made in CSS, and the window should be built in HTML5 canvas. Sounds will uses HTML5 audio. Screen size should be 800 by 600 pixels.

External Interface Requirements

3.1 User Interfaces

The game over screen should have the "Final score" sign as the first line with the actual score below it. Below the score number should be the breakdown of the points earned. Each next line below the score should have the image of the character with number of points it's worth to the right of the image followed by "X" (multiplied by) the number of times the character was hit. Do not calculate.
List the characters from highest worth to lowest.

Example:

Final Score
85
the image of Trustee 10 X 5
the image of Professor 5 X 7

3.2 Hardware Interfaces

The Whack-A-Prof game is going to run in a web browser so it should be able to run on most modern hardware devices including desktops, laptops, and mobile devices. All user input should come from mouse clicks, keyboard strokes, or screen touches (only on hardware where this applies). There is no other special hardware requirements to run the game.


3.3 Software Interfaces

The game will run on a user’s local machine in their desired web browser. Most common browsers include Google Chrome, Mozilla Firefox, and Safari.
